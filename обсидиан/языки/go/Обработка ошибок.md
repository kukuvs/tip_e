#### Подход к обработке ошибок

В Go обработке ошибок уделено особое внимание. В отличие от многих других языков, где ошибки обрабатываются с помощью механизма исключений (exceptions), в Go применяется другой подход, который вносит больше ясности в процесс обработки ошибок.

#### Исключения в других языках

В других языках программирования ошибки обрабатываются с помощью конструкции `try — catch`, которая перехватывает исключения и обрабатывает их:

```python
try:
    foo()
except IndexError:
    # обработка исключений, связанных с выходом за пределы массива
except:
    # обработка всех остальных исключений 
```

Недостатки этого подхода:
- Исключения выбрасываются неявно для вызывающего кода.
- Программисту нужно запоминать, какая функция может выбросить исключение.
- Существуют uncatchable-исключения, например, связанные с выходом программы за пределы доступной ей памяти.

#### Обработка ошибок в Go

Функции в Go могут возвращать больше одного значения. Это позволяет использовать в качестве последнего возвращаемого значения интерфейс `error`:

```go
package main

import "fmt"

func div(a, b int) (int, error) {
    if b == 0 {
        return 0, fmt.Errorf("делитель равен 0")
    }
    return a / b, nil
}

func main() {
    d, err := div(10, 0)
    if err != nil {
        fmt.Println(err)
    } else {
        fmt.Printf("d = %d", d)
    }
}
```

#### Тип `error` в Go

Тип `error` в Go встроенный, то есть, чтобы им воспользоваться, не нужно импортировать какой-либо пакет. Наличие отдельного типа позволяет одинаково обрабатывать ошибки независимо от функции — стандартной или сторонней библиотеки. Вы всегда работаете с одним и тем же интерфейсом `error`, что позволяет сравнивать ошибки или применять функции интроспекции ошибок из библиотеки `errors`.

#### Принудительная обработка ошибок

Поскольку возможность возникновения ошибки при выполнении функции отражена непосредственно в её сигнатуре (последнее возвращаемое значение имеет тип `error`), программист вынужден всегда обрабатывать или игнорировать ошибку явно, иначе код не скомпилируется. Добавлять последним (обычно вторым) возвращаемым аргументом ошибку принято везде, где только может произойти ошибка. Чаще всего речь идет о функциях, в теле которых происходят операции ввода-вывода.
